DATABASE SCHEMA OVERVIEW
================================================================================

TABLES (5 application tables + 1 auth):
  1. auth.users (Supabase) - User authentication (implicit)
  2. profiles - User profile data
  3. categories - Software categories
  4. tools - Directory entries with full-text search
  5. reviews - User ratings and reviews
  6. saved_tools - User bookmarks

ENUMS (3 custom types):
  - user_role: 'user', 'admin', 'moderator'
  - org_size: 'small', 'medium', 'large'
  - pricing_model: 'free', 'freemium', 'nonprofit_discount'

FUNCTIONS (4):
  - update_updated_at() - Auto-update timestamp columns
  - update_tool_rating() - Recalculate tool ratings
  - update_tool_search_vector() - Maintain full-text search
  - handle_new_user() - Auto-create profile on signup

TRIGGERS (6):
  - profiles: timestamp update on modify
  - tools: timestamp update + search vector on modify
  - reviews: rating update on change, timestamp update
  - auth.users: profile creation on new user signup

ROW-LEVEL SECURITY (19 policies):
  - profiles: 3 policies (select all, update own, delete own)
  - categories: 4 policies (select all, admin write/delete)
  - tools: 5 policies (select verified, authenticated submit, admin manage)
  - reviews: 4 policies (select public, authenticated CRUD own)
  - saved_tools: 3 policies (user CRUD own only)

INDEXES (7):
  - tools.search_vector (GIN for full-text search)
  - tools.category_id
  - tools.slug
  - tools.is_verified, is_featured
  - tools.rating_avg DESC
  - reviews.tool_id, reviews.user_id
  - saved_tools.user_id

================================================================================
KEY FEATURES
================================================================================

1. FULL-TEXT SEARCH
   - PostgreSQL tsvector on tools table
   - Combines: name, description, tags, nonprofit_deal
   - Auto-updated on INSERT/UPDATE
   - GIN index for O(1) lookups

2. AUTOMATIC RATING CALCULATION
   - Denormalized rating_avg and review_count on tools
   - Trigger recalculates after every review change
   - No expensive aggregates on query time

3. ROW-LEVEL SECURITY
   - All tables have RLS enabled
   - Prevents unauthorized data access at database level
   - Works with Supabase JWT authentication

4. AUDIT TRAIL
   - created_at on all tables
   - updated_at auto-maintained on modifications
   - approved_at, approved_by track tool verification
   - submitted_by tracks content origin

5. DATA INTEGRITY
   - CASCADE deletes for user-dependent data
   - CHECK constraints on enums and ratings
   - UNIQUE constraints prevent duplicates
   - Foreign key constraints maintain referential integrity

6. FLEXIBLE TOOL SUBMISSION
   - Users can submit unverified tools
   - Admins review and approve
   - Separate verification for "confirmed working"
   - Support for drafts and unpublished tools

================================================================================
SECURITY MODEL
================================================================================

RLS POLICIES SUMMARY:

profiles:
  - SELECT: Public (anyone can view all profiles)
  - UPDATE: Own profile only
  - DELETE: Own profile only

categories:
  - SELECT: Public
  - INSERT/UPDATE/DELETE: Admin role only

tools:
  - SELECT: Verified OR approved OR own submission OR admin/moderator
  - INSERT: Authenticated users (creates unverified)
  - UPDATE: Own unverified tools OR admin/moderator
  - DELETE: Admin/moderator only

reviews:
  - SELECT: Reviews on verified/approved tools
  - INSERT: Authenticated users
  - UPDATE: Own reviews only
  - DELETE: Own reviews only

saved_tools:
  - SELECT: Own bookmarks only
  - INSERT: Own bookmarks only
  - DELETE: Own bookmarks only

AUTHENTICATION:
  - Uses Supabase auth.users table
  - JWT tokens for API authentication
  - auth.uid() function for user identification
  - Automatic profile creation on signup

================================================================================
PERFORMANCE CHARACTERISTICS
================================================================================

QUERY TIMES (approximate on typical data):

  - Get all verified tools: O(log N) with index on is_verified
  - Search by keyword: O(log N) with GIN index on search_vector
  - Get user's reviews: O(log N) with index on reviews.user_id
  - Get tool ratings: O(1) with denormalized rating_avg
  - Filter by category: O(log N) with index on tools.category_id

STORAGE (example with 10K tools, 50K reviews):
  - tools table: ~2-3MB
  - reviews table: ~5-8MB
  - search_vector index: ~2-3MB
  - Total indexes: ~10-15MB

SCALABILITY:
  - Designed for millions of tools and reviews
  - Denormalization strategy avoids N+1 queries
  - Indexes cover all common filter patterns
  - Full-text search efficient for large datasets
